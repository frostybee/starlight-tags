A plugin to add tags to your Starlight documentation site. Tags help organize and categorize your content, making it easier for users to discover related pages.

## Prerequisites

You will need to have a Starlight website set up.
If you don't have one yet, you can follow the ["Getting Started"](https://starlight.astro.build/getting-started) guide in the Starlight docs to create one.

## Installation

import { Steps, Tabs, TabItem } from '@astrojs/starlight/components'

<Steps>

1. `starlight-tags` is a Starlight [plugin](https://starlight.astro.build/reference/plugins/). Install it by running the following command in your terminal:

   <Tabs syncKey="pkg">

   <TabItem label="npm">

   ```sh
   npm install starlight-tags
   ```

   </TabItem>

   <TabItem label="pnpm">

   ```sh
   pnpm add starlight-tags
   ```

   </TabItem>

   <TabItem label="Yarn">

   ```sh
   yarn add starlight-tags
   ```

   </TabItem>

   </Tabs>

2. Configure the plugin in your Starlight [configuration](https://starlight.astro.build/reference/configuration/#plugins) in the `astro.config.mjs` file.

   ```diff lang="js"
   // astro.config.mjs
   import starlight from '@astrojs/starlight'
   import { defineConfig } from 'astro/config'
   +import starlightTags from 'starlight-tags'

   export default defineConfig({
     integrations: [
       starlight({
   +      plugins: [starlightTags()],
         title: 'My Docs',
       }),
     ],
   })
   ```

3. Create a `tags.yml` file in your project root to define your tags:

   ```yaml
   # tags.yml
   tags:
     authentication:
       label: "Authentication"
       description: "API authentication and security"
       color: "#10b981"
       icon: "üîê"

     rest-api:
       label: "REST API"
       description: "RESTful API endpoints"
       color: "#3b82f6"

     stable:
       label: "Stable"
       description: "Production-ready features"
       color: "#22c55e"

   defaults:
     color: "#6b7280"
   ```

   :::note[Example Tags]
   These examples use API/SDK documentation tags, but the same patterns work for any documentation type‚Äîlibraries, frameworks, applications, tutorials, or product guides. Adapt the tag names and descriptions to match your content.
   :::

4. Add tags to your pages using frontmatter:

   ```mdx
   ---
   title: Authentication Guide
   tags:
     - authentication
     - rest-api
     - stable
   ---

   Your content here...
   ```

5. [Start the development server](https://starlight.astro.build/getting-started/#start-the-development-server) to preview the plugin in action.

</Steps>

## What You Get

After installation, the plugin automatically creates:

- **Tags Index Page** at `/tags` - Lists all available tags
- **Tag Detail Pages** at `/tags/[tag]` - Shows all pages for each tag
