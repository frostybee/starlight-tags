---
title: PageWithTags
description: Wrapper component that adds tags to content
---

The `PageWithTags` component wraps your content and automatically displays tags at configurable positions (top, bottom, or both).

## Import

```js
import { PageWithTags } from 'starlight-tags/components';
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `tags` | `ProcessedTag[]` | Required | Array of tag data objects |
| `position` | `'top' \| 'bottom' \| 'both'` | `'bottom'` | Where to display tags |
| `showTitle` | `boolean` | `true` | Show "Tags" heading (bottom only) |

## Basic Usage

```astro
---
import { PageWithTags } from 'starlight-tags/components';

// Get tags for this page
const pageTags = processor.getTagsForPage(pageId);
---

<PageWithTags tags={pageTags}>
  <h1>My Page Title</h1>
  <p>Page content goes here...</p>
</PageWithTags>
```

## Positions

### Bottom (Default)

Tags appear after the content with a separator:

```astro
<PageWithTags tags={pageTags} position="bottom">
  <slot />
</PageWithTags>
```

### Top

Tags appear before the content:

```astro
<PageWithTags tags={pageTags} position="top">
  <slot />
</PageWithTags>
```

### Both

Tags appear at both top and bottom:

```astro
<PageWithTags tags={pageTags} position="both">
  <slot />
</PageWithTags>
```

## Without Title

Hide the "Tags" heading at the bottom:

```astro
<PageWithTags tags={pageTags} showTitle={false}>
  <slot />
</PageWithTags>
```

## Complete Example

```astro
---
import { PageWithTags } from 'starlight-tags/components';
import { TagsProcessor } from 'starlight-tags';
import { getCollection } from 'astro:content';
import { config } from 'virtual:starlight-tagging/config';

// Get page data
const { entry } = Astro.props;
const frontmatterTags = entry.data.tags || [];

// Initialize processor
const logger = { info: () => {}, warn: () => {}, error: () => {} };
const docs = await getCollection('docs');
const processor = new TagsProcessor(config, logger, docs);
await processor.initialize();

// Get processed tag objects
const pageTags = frontmatterTags
  .map(tagId => processor.getTag(tagId))
  .filter(Boolean);
---

<PageWithTags tags={pageTags} position="both" showTitle={true}>
  <article>
    <h1>{entry.data.title}</h1>
    <slot />
  </article>
</PageWithTags>
```

## Styling

The component adds margin and border styling:

- **Top position**: Margin below tags
- **Bottom position**: Margin above tags with a top border separator

### Custom Styling

Override the default styles:

```css
.page-tags {
  margin: 1rem 0;
}

.page-tags--top {
  margin-bottom: 2rem;
}

.page-tags--bottom {
  border-top: 2px solid var(--sl-color-accent);
  padding-top: 1rem;
}
```

## When to Use

Use `PageWithTags` when you need to:

- Control tag display at the layout level
- Show tags in a consistent position across multiple pages
- Have more control over tag data (e.g., filtering, custom ordering)

For simpler use cases where you just want to show the current page's tags, consider using [`PageTags`](/components/page-tags) instead.

## Components vs Auto-Generated Pages

It's important to understand the difference between **components** you use in your content and the **pages** automatically generated by the plugin:

| Aspect | `PageWithTags` (Component) | `/tags` Index Page |
|--------|---------------------------|-------------------|
| **Type** | Component you import | Auto-generated route |
| **Purpose** | Wrap content with tags | List all tags in system |
| **Shows** | Tags for a single page | All tags with stats & cloud |
| **Data** | You provide pre-processed tags | Fetches all tags automatically |
| **Localization** | Hardcoded title (customizable) | Fully localized UI |
| **Usage** | `<PageWithTags tags={...}>` | Visit `/tags` URL |

### Component (PageWithTags)

A wrapper you control - add it to your pages or layouts to display tags around content:

```astro
<PageWithTags tags={pageTags} position="bottom">
  <slot />
</PageWithTags>
```

### Auto-Generated Pages

The plugin automatically creates these routes:
- **`/tags`** - Index page listing all tags with stats, tag cloud, and alphabetical navigation
- **`/tags/[tag-slug]`** - Individual tag pages showing all pages with that tag

These pages are fully localized and styled - you don't need to create them yourself.
