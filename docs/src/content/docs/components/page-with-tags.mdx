title: PageWithTags
description: Wrapper component that adds tags to content
---

The `PageWithTags` component wraps your content and automatically displays tags at configurable positions (top, bottom, or both).

**[See live demo →](/demos/page-with-tags)**

:::tip[Wrapper component with positioning control]
```astro
<PageWithTags tags={tags} position="both">
  <YourContent />
</PageWithTags>
```

- **Requires tags prop** - you must pass pre-fetched `ProcessedTag[]` data
- **Wraps content** via `<slot />` - your content goes inside
- **Position control** - `top`, `bottom`, or `both`
- Adds separator line and "Tags" title for bottom position
- Best for: Layout templates where you want tags to frame content
:::

## Import

```js
import { PageWithTags } from 'starlight-tags/components';
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `tags` | `ProcessedTag[]` | Required | Array of tag data objects |
| `position` | `'top' \| 'bottom' \| 'both'` | `'bottom'` | Where to display tags |
| `showTitle` | `boolean` | `true` | Show "Tags" heading (bottom only) |

## Basic Usage

```astro
---
import { PageWithTags } from 'starlight-tags/components';
import { initTags, getTagsForPage } from 'virtual:starlight-tagging/tags';

// Initialize tags store (memoized - safe to call multiple times)
await initTags();

// Get tags for this page
const pageTags = getTagsForPage('guides/authentication');
---

<PageWithTags tags={pageTags}>
  <h1>My Page Title</h1>
  <p>Page content goes here...</p>
</PageWithTags>
```

## Positions

### Bottom (Default)

Tags appear after the content with a separator:

```astro
<PageWithTags tags={pageTags} position="bottom">
  <slot />
</PageWithTags>
```

### Top

Tags appear before the content:

```astro
<PageWithTags tags={pageTags} position="top">
  <slot />
</PageWithTags>
```

### Both

Tags appear at both top and bottom:

```astro
<PageWithTags tags={pageTags} position="both">
  <slot />
</PageWithTags>
```

## Without Title

Hide the "Tags" heading at the bottom:

```astro
<PageWithTags tags={pageTags} showTitle={false}>
  <slot />
</PageWithTags>
```

## Complete Example

```astro
---
import { PageWithTags } from 'starlight-tags/components';
import { initTags, getTag } from 'virtual:starlight-tagging/tags';

// Get page data
const { entry } = Astro.props;
const frontmatterTags = entry.data.tags || [];

// Initialize tags store (memoized - safe to call multiple times)
await initTags();

// Get processed tag objects
const pageTags = frontmatterTags
  .map(tagId => getTag(tagId))
  .filter(Boolean);
---

<PageWithTags tags={pageTags} position="both" showTitle={true}>
  <article>
    <h1>{entry.data.title}</h1>
    <slot />
  </article>
</PageWithTags>
```

## Styling

The component adds margin and border styling:

- **Top position**: Margin below tags
- **Bottom position**: Margin above tags with a top border separator

### Custom Styling

Override the default styles:

```css
.page-tags {
  margin: 1rem 0;
}

.page-tags--top {
  margin-bottom: 2rem;
}

.page-tags--bottom {
  border-top: 2px solid var(--sl-color-accent);
  padding-top: 1rem;
}
```

## PageWithTags vs PageTags

Wondering which component to use? Here's a quick comparison:

| Feature | `PageWithTags` | `PageTags` |
|---------|----------------|-----------|
| Auto-reads frontmatter | ❌ No | ✅ Yes |
| Requires tags prop | ✅ Yes | ❌ No |
| Wraps content | ✅ Yes (via slot) | ❌ No |
| Position control | ✅ `top` / `bottom` / `both` | ❌ Manual placement |
| Separator line | ✅ Yes (bottom position) | ❌ No |
| "Tags" title | ✅ Optional | ❌ No |
| Best for | Layout templates | Drop-in tag display |

### When to Use `PageWithTags`

Choose `PageWithTags` when you need:

- **Content wrapping** - Tags frame your content automatically
- **Consistent positioning** - Enforce same layout across pages (great for layout components)
- **Custom tag data** - You control which tags display, with filtering or custom ordering
- **Visual separation** - Built-in separator line and optional "Tags" title

```astro
<PageWithTags tags={pageTags} position="both">
  <article>
    <slot />
  </article>
</PageWithTags>
```

### When to Use `PageTags`

Choose [`PageTags`](/components/page-tags) when you want:

- **Simplicity** - Just drop `<PageTags />` anywhere in your MDX
- **Flexibility** - Place tags exactly where you want in your content
- **No setup** - Automatically reads tags from the page's frontmatter

```mdx
<PageTags />  <!-- That's it! -->
```

## Components vs Auto-Generated Pages

It's important to understand the difference between **components** you use in your content and the **pages** automatically generated by the plugin:

| Aspect | `PageWithTags` (Component) | `/tags` Index Page |
|--------|---------------------------|-------------------|
| **Type** | Component you import | Auto-generated route |
| **Purpose** | Wrap content with tags | List all tags in system |
| **Shows** | Tags for a single page | All tags with stats & cloud |
| **Data** | You provide pre-processed tags | Fetches all tags automatically |
| **Localization** | Hardcoded title (customizable) | Fully localized UI |
| **Usage** | `<PageWithTags tags={...}>` | Visit `/tags` URL |

### Component (PageWithTags)

A wrapper you control - add it to your pages or layouts to display tags around content:

```astro
<PageWithTags tags={pageTags} position="bottom">
  <slot />
</PageWithTags>
```

### Auto-Generated Pages

The plugin automatically creates these routes:
- **`/tags`** - Index page listing all tags with stats, tag cloud, and alphabetical navigation
- **`/tags/[tag-slug]`** - Individual tag pages showing all pages with that tag

These pages are fully localized and styled - you don't need to create them yourself.
