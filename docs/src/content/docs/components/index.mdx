The `starlight-tags` plugin exports several Astro components for displaying tags in your documentation pages. All components can be imported from `starlight-tags/components`.

:::note
The examples on this page use API/SDK documentation scenarios. These patterns apply to any documentation type.
:::

## Available Components

| Component | Purpose |
|-----------|---------|
| [`TagBadge`](/components/tag-badge) | Display a single tag badge |
| [`TagsList`](/components/tags-list) | Display a list of tags |
| [`PageTags`](/components/page-tags) | Auto-display tags for the current page |
| [`PageWithTags`](/components/page-with-tags) | Wrapper that adds tags to content |
| [`PrerequisiteChain`](/components/prerequisite-chain) | Display prerequisite learning path |

## Importing Components

```mdx
---
title: Authentication Guide
tags:
  - authentication
  - rest-api
---

import { PageTags, TagBadge, TagsList } from 'starlight-tags/components';

# Authentication Guide

<PageTags />

Content here...
```

## Quick Examples

### Display Page Tags

The simplest way to show tags on a page:

```mdx
import { PageTags } from 'starlight-tags/components';

<PageTags />
```

### Custom Tag Badge

Display a specific tag with custom styling:

```astro
---
import { TagBadge } from 'starlight-tags/components';
import { initTags, getTag } from 'virtual:starlight-tagging/tags';

await initTags();
const tag = getTag('authentication');
---

<TagBadge tag={tag} variant="outline" size="large" />
```

### Tags List

Display multiple tags with sorting:

```astro
---
import { TagsList } from 'starlight-tags/components';
import { initTags, getAllTagsSorted } from 'virtual:starlight-tagging/tags';

await initTags();
const tags = getAllTagsSorted();
---

<TagsList tags={tags} layout="grid" showCount={true} />
```

## Component Props

Each component accepts different props for customization:

### Common Props

| Prop | Description |
|------|-------------|
| `variant` | Visual style: `'solid'`, `'outline'`, or `'ghost'` |
| `size` | Size: `'small'`, `'medium'`, or `'large'` |
| `showIcon` | Whether to show tag icons |
| `showCount` | Whether to show page counts |

### Style Variants

All tag components support three visual variants:

- **`solid`** - Filled background with tag color
- **`outline`** - Bordered with transparent background
- **`ghost`** - Subtle background tint

## Getting Tag Data

Components that display specific tags need `ProcessedTag` data. Use the virtual module for efficient, memoized access:

```astro
---
import { initTags, getTag, getAllTagsSorted } from 'virtual:starlight-tagging/tags';

// Initialize once (memoized - safe to call multiple times)
await initTags();

// Get a single tag
const authTag = getTag('authentication');

// Get all tags sorted by popularity
const allTags = getAllTagsSorted();
---
```

The `initTags()` function is memoized, so calling it multiple times (e.g., in multiple components on the same page) only initializes once. This ensures efficient builds regardless of how many components use tag data.
