The `starlight-tags` plugin exports several Astro components for displaying tags in your documentation pages. All components can be imported from `starlight-tags/components`.

:::tip[Self-contained components]
All components handle data fetching automatically. Just pass a `tagId` or `tagIds` prop and they fetch the tag data for you. No manual data fetching required!
:::

## Available Components

| Component | Purpose |
|-----------|---------|
| [`TagBadge`](/starlight-tags/components/tag-badge/) | Display a single tag badge |
| [`TagsList`](/starlight-tags/components/tags-list/) | Display a list of tags |
| [`PageTags`](/starlight-tags/components/page-tags/) | Auto-display tags for the current page |
| [`PageWithTags`](/starlight-tags/components/page-with-tags/) | Wrapper that adds tags to content |
| [`PrerequisiteChain`](/starlight-tags/components/prerequisite-chain/) | Display prerequisite learning path |

## Importing Components

```mdx
---
title: Authentication Guide
tags:
  - authentication
  - rest-api
---

import { PageTags, TagBadge, TagsList } from 'starlight-tags/components';

# Authentication Guide

<PageTags />

Content here...
```

## Quick Examples

### Display Page Tags

The simplest way to show tags on a page:

```mdx
import { PageTags } from 'starlight-tags/components';

<PageTags />
```

### Single Tag Badge

Display a specific tag:

```astro
import { TagBadge } from 'starlight-tags/components';

<TagBadge tagId="authentication" />
<TagBadge tagId="rest-api" variant="outline" size="large" />
```

### Tags List

Display multiple tags:

```astro
import { TagsList } from 'starlight-tags/components';

<!-- Show all tags -->
<TagsList />

<!-- Show specific tags -->
<TagsList tagIds={['authentication', 'webhooks', 'rest-api']} />

<!-- With options -->
<TagsList layout="grid" showCount limit={6} />
```

### Prerequisite Chain

Show learning path for a topic:

```astro
import { PrerequisiteChain } from 'starlight-tags/components';

<PrerequisiteChain tagId="oauth2" />
```

## Component Props

Each component accepts different props for customization:

### Common Props

| Prop | Description |
|------|-------------|
| `tagId` / `tagIds` | Tag ID(s) to display (fetches data automatically) |
| `variant` | Visual style: `'solid'`, `'outline'`, or `'ghost'` |
| `size` | Size: `'small'`, `'medium'`, or `'large'` |
| `showIcon` | Whether to show tag icons |
| `showCount` | Whether to show page counts |

### Style Variants

All tag components support three visual variants:

- **`solid`** - Filled background with tag color
- **`outline`** - Bordered with transparent background
- **`ghost`** - Subtle background tint

## Advanced: Manual Data Fetching

For advanced use cases (custom filtering, caching, etc.), you can fetch tag data manually:

```astro
---
import { TagsList } from 'starlight-tags/components';
import { initTags, getAllTagsSorted } from 'virtual:starlight-tagging/tags';

await initTags();
const allTags = getAllTagsSorted();
const popularTags = allTags.filter(tag => tag.count >= 3);
---

<TagsList tags={popularTags} />
```
